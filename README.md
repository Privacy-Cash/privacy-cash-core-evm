# Privacy Cash Core EVM

Privacy pool for ETH transactions using zero-knowledge proofs.

## Setup

```bash
yarn install
yarn compile
```

## Environment

Create a `.env` file:

```
PRIVATE_KEY=<deployer private key>
ALCHEMY_KEY=<alchemy api key>
ETHERSCAN_KEY=<etherscan/basescan api key>
ADMIN_ADDRESS=<admin/multisig address (optional, defaults to 0x44eb9939cfdE7C394f1632C6890191d695f0a3ce)>
```

## Tests

```bash
yarn test
```

## Deploy (Base Sepolia)

```bash
npx hardhat run scripts/deployTestnet.js --network baseSepolia
```

## Upgrade

Deploy a new implementation and upgrade the proxy:

```bash
PROXY_ADDRESS=<proxy address> npx hardhat run scripts/deployUpgrade.js --network baseSepolia
```

If the deployer is the admin, the upgrade executes immediately. If the admin is a multisig, the script deploys and validates the new implementation, then the multisig must call `upgradeToAndCall(newImpl, "0x")` on the proxy.

## Architecture

- **EtherPool** -- Main contract handling deposits, withdrawals, and shielded transfers with ZK proof verification. Inherits from `MerkleTreeWithHistory` and `UUPSUpgradeable`.
- **MerkleTreeWithHistory** -- Poseidon-based Merkle tree with a circular root history buffer (100 roots).
- **Verifier2** -- Groth16 ZK proof verifier (generated by snarkjs).
- **Proxy** -- ERC-1967 UUPS proxy. Upgrade authority is controlled by the contract's `admin`.
